## 数据流

### 1. 用户认证流程
```
用户登录 → Supabase Auth → 创建 session → middleware 检测 →
  已登录 → 进入仪表盘
  未登录 → 重定向到登录页
```

### 2. 审图规划流程
```
用户创建项目 → 上传设计说明文件 → 点击"生成审图规划" →
  API Route 调用 DeepSeek → 解析文件内容 →
  生成审查要点清单 → 保存到 review_plans + plan_items 表 →
  前端展示
```

### 3. 规范查询流程
```
用户输入问题 → API Route 调用 Coze 工作流 regulation_search_rag →
  工作流返回规范条文 → 保存到 code_queries 表 →
  前端展示
```

### 4. 专家匹配流程
```
用户输入/选择问题 → API Route 调用 Coze 工作流 Master_Match →
  工作流返回 TOP-3 专家推荐 → 保存到 expert_matches 表 →
  前端展示
```

### 5. 知识库上传流程（管理员）
```
管理员上传文件 → API Route 验证角色(admin) →
  上传文件到存储 → 同步到 Coze 知识库 →
  更新 knowledge_base_files 表状态 →
  前端反馈
```

## 模块划分

### MVP 功能模块
| 模块       | 前端页面                                  | API/后端逻辑                        | 数据库表               |
| ---------- | ----------------------------------------- | ----------------------------------- | ---------------------- |
| 项目创建   | 项目列表 + 新建项目 + 文件上传             | Supabase CRUD, 文件存储              | projects, project_files|
| 审图规划   | 审查要点清单页                             | DeepSeek API + LangChain            | review_plans, plan_items|
| 规范查询   | 对话式查询页                               | Coze 工作流代理                      | code_queries           |
| 专家匹配   | 专家推荐结果页                             | Coze 工作流代理                      | expert_matches         |
| 知识库上传 | 上传管理页（管理员）                       | 文件存储 + Coze 知识库同步            | knowledge_base_files   |

### 非 MVP 占位模块
- 报告生成
- 报告对比
- 问题统计
- 项目管理
- 知识库查询

## 安全边界

1. **API Key 不暴露**: DeepSeek Key、Coze Token、Supabase Service Role Key 仅存在于服务端 API Routes 和 .env 中
2. **前端只使用 Anon Key**: Supabase 的 Anon Key 公开但受 RLS 策略保护
3. **角色校验**: 管理员操作在 API Route 中二次验证用户角色
4. **RLS 行级安全**: 用户只能访问自己的数据，管理员只能管理知识库
