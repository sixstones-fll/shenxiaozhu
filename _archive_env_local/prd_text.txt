=== PRD 内容提取 ===
[Normal] Demo PRD-审小助
[Normal] 1. 行业-客户-业务分析
[Normal] 行业(Industry):建筑设计行业-施工图设计
[Normal] 业务(Business)：建筑工程施工图审查助手
[Normal] 2. 角色分类
[Normal] 审图计划制定
[Normal] 专业负责人：分析项目特点，厘清建筑类型、审查属性；整理适配国家规范、梳理强条，补充高频错漏与项目专属风险点，输出审查标准提交项目负责人审定
[Normal] 项目负责人：审定专业负责人提交的项目审查标准与核查重点
[Normal] 图纸问题审查
[Normal] 图审人员：依据审查标准逐张逐项审图，定位分级违规、专业冲突问题，标注规范条文，生成《审图报告》；输出整改建议，复核图纸整改情况
[Normal] 设计师：对接图审人员获取整改意见，接收审图问题清单
[Normal] 违规问题整改
[Normal] 设计师：解读《审图报告》与整改建议，对照规范修改图纸并提交整改回复，多轮提交复查直至图纸无误定稿
[Normal] 图审人员：对整改后图纸开展复查，校验问题是否闭环、有无新增违规问题
[Normal] 项目归档复盘
[Normal] 专业负责人：统筹全套图纸、审图全流程资料整理归档
[Normal] 项目负责人：牵头项目复盘，提炼项目高频错漏、典型问题
[Normal] 3. 技术选择
[Normal] 3.1 业务边界
[Normal] 本系统聚焦于建筑工程图纸审查智能化场景，涵盖审图要点清单规划、规范智能检索、业务专家智能匹配、审图报告智能生成与对比、整改任务全生命周期追踪。 系统不涉及CAD图纸内容识别，不处理工程采购成本核算、施工工程量计算、项目人员考勤、现场施工进度管控、工程造价计价等非图纸审图类业务诉求。
[Normal] 3.2 核心目标
[Normal] 通过大语言模型（LLM）与检索增强生成（RAG）多知识库检索技术，整合规范知识库、历史审图问题库、历史项目审图报告库，将传统审图 “人工制定审图范围、人工翻规范、人工复盘同类问题、人工对接专业能手” 的低效模式，转变为审查清单自动匹配、AI 规范智能检索、智能匹配专家的智能化审图模式；大幅降低建筑、结构、机电、消防审图工程师的规范查阅、报告撰写负荷，缩短图纸问题研判、整改、复核全流程周期。
[Normal] 3.3 数据库
[Normal] 规范知识库分为册级和条文级，册级按专业分，只保留规范名称；条文级别按专业分，每个规范下面的每条规范单条进行保存。历史问题库的每条问题记录问题描述、问题违反的规范条文、所属项目、设计人、图审人、建筑类型、问题强条类型。
[Normal] 审图规划链路：识别生成项目信息（必须识别的信息包括名称、编号，建筑类型、防火分类、结构形式、抗震设防烈度等核心建筑属性，人防 / 绿建 / 特殊功能等专项设计信息），用项目关键词去搜索规范库-册级，（eg:如果有人防就返回《人民防空地下室设计规范》），用关键词检索问题库里的问题，返回TOP 3高频问题。融合后通过大模型输出《审查要点清单》，右侧面板提供预览；最终一键生成并下载完整清单文件。
[Normal] 规范问答链路：用户问题关键词提取，在规范库-条文级里检索相关的规范条文，返回精简解读内容，点击对话卡片内「查看完整规范」，右侧弹出详情面板展示规范编号、原文表格、审查要点、版本对比信息，支持逐条查阅完整条文详情。
[Normal] 专家匹配链路：用户问题+项目特征提取，根据项目特征（建筑类型、专业）和问题特征搜索历史问题库，召回同类项目的相似问题，定位问题关联的图审人和设计师，把TOP-3的两类人员召回，去重排序去前三名。
[Normal] 4. 功能清单
[Normal] 功能名称1：项目创建
[Normal] 技术实现：用户会话绑定。
[Normal] 功能详情：提供统一的任务入口。用户点击「新建项目」创建项目,系统自动建立独立项目工作区。在这个界面直接上次项目设计说明文件。
[Normal] 功能价值：解决多项目任务无隔离、各功能模块数据不互通的痛点，以项目为载体统一承载全部业务操作，无需重复录入项目信息，简化跨功能切换操作。
[Normal] 核心目标：快速隔离不同审图项目数据，为各项功能提供统一项目上下文与数据隔离环境。
[Normal] 考虑因素：项目基础字段校验、同用户下项目名称唯一校验；当前项目状态本地缓存，页面刷新自动恢复选中项目；多项目并行操作的数据隔离机制。
[Normal] 功能名称 2：审图规划
[Normal] 技术实现：审查规划 Agent 调度、项目特征提取大模型解析、多维度 Query 拆分、BM25 关键词检索 + Milvus 向量多路召回、RRF 分数融合排序、Reranker 重排、大模型归纳生成规范清单与高频问题、前端表单联动渲染、文件导出流处理。
[Normal] 功能详情：切换顶部「审图规划」标签进入功能页，根据用户上传项目文件，大模型识别生成项目信息（必须识别的信息包括名称、编号，建筑类型、防火分类、结构形式、抗震设防烈度等核心建筑属性，人防 / 绿建 / 特殊功能等专项设计信息），支持用户确认或重新勾选，支持用户自定义补充审图约束条件；确认后点击提交；用项目关键词去搜索规范库-册级，（eg:如果有人防就返回《人民防空地下室设计规范》）；同时用关键词检索问题库里的问题，返回TOP 3高频问题。融合后通过大模型输出《审查要点清单》，右侧面板提供预览；最终一键生成并下载完整清单文件。
[Normal] 功能价值：解决专业负责人人工梳理适配规范、提炼核查要点耗时久、历史高频错漏无法自动复用的痛点，基于项目业态自动生成标准化审图依据与核查清单，统一全院审图规划标准。
[Normal] 核心目标：依托项目属性自动输出结构化、可导出的审查要点清单，为审图计划制定提供标准化、全覆盖的核查依据。
[Normal] 功能名称3：规范查询
[Normal] 技术实现：用户对话文本分词与意图识别、RAG 规范知识库向量检索、BM25 关键词混合召回、大模型条文摘要生成、侧边弹窗详情页渲染、对话历史持久化存储。
[Normal] 功能详情：切换顶部「规范问答」标签进入对应功能，用户在底部输入框提交规范相关问题发起对话检索；系统进行用户问题关键词提取，在规范库-条文级里检索相关的规范条文，返回精简解读内容；点击对话卡片内「查看完整规范」，右侧弹出详情面板展示规范编号、原文表格、审查要点、版本对比信息，支持逐条查阅完整条文详情。
[Normal] 功能价值：解决人工翻阅规范书籍检索慢、条文解读零散、强条要点不易提炼的痛点，通过对话式检索快速定位匹配条文，配套完整原文与审图重点，大幅降低规范查阅与理解成本。
[Normal] 核心目标：依托规范知识库，实现自然语言一键检索对应国标条文，自动提炼适配项目的审查要点。
[Normal] 考虑因素：对话输入多意图兼容识别；向量库检索召回准确率与检索速度平衡；规范详情弹窗表格、备注要点自适应排版；单项目对话历史绑定存储、切换项目自动隔离对话记录；规范新旧版本差异自动对比展示逻辑。
[Normal] 功能名称 3：专家匹配
[Normal] 技术实现：Query 提取大模型语义解析、专家匹配 Agent 调度、BM25 关键词检索 + Milvus 向量多路召回、RRF 分数融合重排、多权重二次排序、大模型生成专家清单与咨询话术、对话交互面板渲染。
[Normal] 功能详情：用户在规范问答输入框输入带专业咨询诉求的问题，系统通过大模型拆分出问题特征，结合之前提取的项目特征双维度检索条件；根据项目特征（建筑类型、专业）和问题特征搜索历史问题库，召回同类项目的相似问题，定位问题关联的图审人和设计师，把TOP-3的两类人员召回，去重排序去前三名；大模型输出《专家推荐名单》，清晰标注专家属于图审、设计的哪一类；点击对话内入口右侧弹出专家面板，展示职称、同类项目履历、联系方式；点击一键私信，AI 结合当前项目与问题自动生成完整咨询话术，支持单发或群发专家。
[Normal] 功能价值：解决审图疑难问题找不到对口资深人员、人工筛选专家耗时长、线上咨询无规范沟通话术的痛点，依托历史项目与审图记录精准匹配高适配专家，简化跨角色咨询协同流程。
[Normal] 核心目标：基于问题自动筛选同类项目经验丰富、手头负荷适中的专家。
[Normal] 考虑因素：大模型精准拆分项目与问题检索关键词；多路召回 RRF 融合排序权重配比；专家二次排序平衡匹配度与在研项目数量；区分图审 / 设计两类专家分流展示；咨询话术自动带入当前项目上下文。
[Normal] 功能名称 5：报告生成（非MVP）
[Normal] 技术实现：文本结构化抽取大模型、字段完整性校验逻辑、问题标准化改写大模型、审图报告模板渲染引擎、前端表格 CRUD 交互、附件截图上传存储、文件导出流处理、项目问题清单持久化存储、多条件筛选查询。
[Normal] 功能详情：切换顶部「报告生成」标签进入审查问题清单页面，用户可在底部输入框描述图纸问题，点击按钮转写生成标准化问题描述；也可直接在表格手动新增、编辑、删除问题条目；大模型自动抽取图号、图名、违规条文、专业、严重级等结构化字段，缺失信息引导用户补齐；系统标准化改写每条问题记录，完整数据生成结构化《审图报告》；支持人工核查修改内容、上传图纸截图附件，可按专业、严重级多条件筛选问题，最终一键导出完整审图报告文件。
[Normal] 功能价值：解决人工手写审图问题描述口径混乱、逐条整理报告重复工作量大、图纸截图与问题记录分散难归档的痛点，自动标准化问题话术、统一报告格式，大幅减少报告撰写与整理耗时。
[Normal] 核心目标：将自然语言描述的图纸违规问题自动转化为规范结构化审图条目，整合生成符合设计院标准的完整审图意见书，可追溯。
[Normal] 考虑因素：大模型精准拆分多维度结构化字段；问题描述改写统一行业标准话术；表格实时编辑与数据缓存；多维度筛选条件联动查询；导出报告版式适配院内归档模板；截图附件与对应问题绑定存储。
[Normal] 功能名称 6：报告对比（非MVP）
[Normal] 技术实现：报告对比 Agent 调度、文档文本解析、LLM 结构化问题提取、问题唯一匹配 Key 生成、语义相似度打分模型、阈值分类判定逻辑、变更状态大模型划分、多版本报告对象存储、前端分栏渲染、对比汇总报表导出流处理。
[Normal] 功能详情：切换顶部「报告对比」标签进入页面，支持上传新版本、旧版本审图报告文件，也可直接调取项目内已归档历史报告版本；系统解析两份报告，通过大模型提取每条问题的项目部位、构件类型、违规规范编号生成匹配 Key；基于 Key 精准匹配后做语义相似度打分，按阈值自动划分四类变更状态：已整改问题、遗留问题、新增问题、难以判断；页面分栏展示新旧报告全文，右侧面板汇总各类问题数量，点击可查看每条对比详情，支持一键下载完整对比汇总文件，完成对比后两版报告自动归档至当前项目。
[Normal] 功能价值：解决多轮图纸整改后人工逐条核对新旧报告效率低、漏查遗留违规、新增隐患无法快速识别的痛点，AI 自动区分问题整改状态，量化复盘整改效果，降低跨版本复核工作量。
[Normal] 核心目标：自动比对两轮审图报告内全部违规问题，精准识别已整改、遗留、新增、模糊四类问题，直观呈现图纸整改闭环效果。
[Normal] 考虑因素：多格式报告文本解析容错；语义相似度阈值适配建筑审图场景；一对多 / 多对多问题匹配逻辑；前端分栏左右对照展示；四类问题标签筛选查看；导出文件完整保留每条问题原文与对比依据。
[Normal] 功能名称 7：问题统计（非MVP）
[Normal] 技术实现：项目问题全量数据聚合查询、多维度指标分组统计、可视化图表渲染引擎、关键词向量检索、对话语义问题定位、专业人员信息关联查询、前端看板实时数据刷新、问题 ID 关联详情路由跳转。
[Normal] 功能详情：切换顶部「问题统计」标签进入可视化看板页面，系统自动聚合当前项目全部审图问题数据，分四大维度展示统计结果：问题整改状态（总数 / 已整改 / 未整改）、强条风险等级（一类强条 / 二类强条 / 普通问题）、各专业问题分布、各专业对应负责人及联系方式；支持顶部搜索框关键词检索、底部对话输入框自然语言描述检索，快速匹配并定位对应单条问题，点击可跳转查看完整问题详情。
[Normal] 功能价值：解决无法直观掌握全项目审图整改全貌、跨专业问题分布不清晰、高风险强条无集中汇总、查找单条问题需翻阅全部报告的痛点，一站式可视化展示项目审图全局数据，快速定位重点风险与待整改事项。
[Normal] 核心目标：整合项目全生命周期审图问题数据，以多维度可视化看板直观呈现，为项目进度管控、风险前置管控提供量化数据支撑。
[Normal] 考虑因素：多维度统计指标实时同步最新整改数据；强条风险等级醒目区分展示；专业负责人信息与对应问题自动绑定；关键词 + 语义混合检索匹配精度；看板卡片自适应布局；检索结果快速跳转至问题原始记录页面。
[Normal] 功能名称 8：项目管理（非MVP）
[Normal] 技术实现：用户项目数据分库隔离存储、项目名称 / 编号关键词检索、建筑类型标签筛选、侧边详情面板数据渲染、项目全流程操作记录聚合查询、项目软删除逻辑、页面分页列表渲染、项目上下文快速切换路由。
[Normal] 功能详情：点击左侧菜单栏「项目管理」进入页面，分为「我建立的」「历史项目库」两个标签；顶部支持输入项目名称、编号关键词检索，同时提供建筑类型快捷分类筛选；列表展示全部项目名称，每条项目支持查看、删除操作；选中项目后右侧面板展示项目基础信息（建筑类型、创建时间）、历史操作记录（报告生成次数、报告对比次数），点击「进入项目详情」可快速切换至目标项目工作区，加载该项目全部历史对话与审图成果。
[Normal] 功能价值：解决多项目并行、历史项目难以快速检索调取、项目操作记录分散无汇总的痛点，集中管理全部审图项目，快速定位、复用过往项目完整审图资料。
[Normal] 核心目标：统一收纳用户创建的全部审图项目，提供多维度检索筛选能力，实现任意历史项目一键切换访问。
[Normal] 考虑因素：不同用户项目数据权限隔离；关键词模糊检索匹配项目名称与编号；多建筑类型标签快速过滤；项目删除做软删除保留归档数据；右侧详情面板实时加载项目统计数据；切换项目时完整隔离不同项目对话与成果缓存。
[Normal] 功能名称 9：知识库上传（非MVP）
[Normal] 技术实现：多格式文件分片上传、MIME 文件白名单校验、网页文本爬虫抓取、文档多级标题解析、三种文本分段算法、文本预处理清洗逻辑、向量分片嵌入计算、知识库分库存储、三步式前端向导交互。
[Normal] 功能详情：点击左侧菜单栏「知识库」，切换至「上传知识」标签，分2步完成知识库导入；第一步数据导入，选择文档 / 结构化数据类型，选定目标知识库（审图报告库 / 规范库 / 历史问题库），支持本地上传 PDF/TXT/DOC/DOCX/HTML 文件或网络链接抓取网页文本；第二步数据配置，可选自动分段、自定义分段、按标题等级分段三种模式，配置标题层级深度、分段长度、重叠窗口、文本清洗预处理规则；完成配置后进入导入完成步骤，系统自动切片、向量化存入对应知识库，供给全平台检索引擎调用。
[Normal] 功能价值：解决企业规范、历史审图报告、高频问题资料分散在本地文件，无法被 AI 检索复用的痛点，支持批量结构化导入行业审图资料，构建专属行业向量知识库，提升规范匹配、案例检索精准度。
[Normal] 核心目标：支持多渠道、多格式导入建筑审图规范、历史审图报告、历史问题案例，批量构建企业私有审图知识库。
[Normal] 考虑因素：本地文件拖拽上传与分片断点续传；网页爬虫防反爬、文本降噪；多级标题精准识别分层切片；分段长度与重叠窗口参数适配建筑长文档；URL、冗余空白自动清洗；三大知识库物理隔离存储；导入进度实时展示、失败文件单独提示。
[Normal] 功能名称 10：知识库查询（非MVP）
[Normal] 技术实现：多知识库分库路由查询、关键词模糊检索、建筑类型 / 专业标签过滤、分页列表渲染、知识库元数据读取、详情弹窗内容加载、知识库条目编辑 / 删除接口、字母快速索引缓存。
[Normal] 功能详情：点击左侧菜单栏「知识库」，切换对应子标签（审图报告库 / 规范库 / 历史问题库）进入查询列表；顶部输入框支持检索项目名称、规范编号、规范名称，提供首字母快捷检索与分类标签筛选（建筑业态 / 专业门类）；列表分页展示知识库条目，审图报告库可展开查看项目全部历史报告版本，每条条目支持查看详情、编辑、删除操作，点击查看详情可完整读取入库文档分段内容。
[Normal] 功能价值：解决规范、历史审图报告、历史问题分散存储、难以快速定位调取的痛点，分库分类管理全部企业审图知识资产，多维度检索快速定位目标资料，支持在线维护知识库内容。
[Normal] 核心目标：提供三大知识库统一查询入口，实现企业审图知识资产可视化管理。
[Normal] 考虑因素：三大知识库数据物理隔离，切换标签自动切换检索数据源；关键词同时匹配标题、编号、文档正文；建筑业态 / 专业标签多条件组合筛选；分页加载海量知识库条目；报告多版本层级展开渲染；编辑删除操作增加权限二次确认；首字母索引预加载提升检索响应速度。
